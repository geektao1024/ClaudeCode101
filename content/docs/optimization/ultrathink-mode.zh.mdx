# Ultrathink Mode

在 Claude Code prompt 里加入 `ultrathink`，适合需要更深推理的架构设计、复杂调试、迁移方案和高风险实现计划。它可能提升方案质量，但会消耗更多 token，不应该作为日常小改动的默认写法。

## 快速答案

当“错误方案的成本”高于“多花一点推理成本”时，用 `ultrathink`。典型场景包括多文件重构、支付或认证改造、数据迁移、生产故障排查和 Plan Mode 方案审查。

简单文案、格式化、明显 bug 修复、普通 CRUD，不需要 `ultrathink`。

## 适合场景

**适合用：**

- 复杂架构设计
- 技术方案取舍
- 隐蔽 bug 调试
- 多文件迁移方案
- 支付、认证、数据库、重定向等高风险改动

**不适合用：**

- 简单 CRUD
- 样式微调
- 快速 bug 修复
- 格式化
- 明确的一行文案修改

## 和 Plan Mode 搭配

更稳的用法是：先用 `ultrathink` 做方案判断，再用 Plan Mode 审查实现路径，最后再允许 Claude Code 修改文件。

这种方式适合生产仓库、迁移、SEO URL 调整、认证、支付和数据库相关任务。它不是为了显得“更强”，而是为了减少错误方向导致的返工。

## 和 Ultracode 的区别

`ultrathink` 更像单轮 prompt 里的深度推理请求。Ultracode 更适合范围明确的仓库级任务，例如审计、迁移、多文件实现或需要更高 effort 的工作流。

不要把二者都当默认模式。任务越小，越应该用普通模式。
