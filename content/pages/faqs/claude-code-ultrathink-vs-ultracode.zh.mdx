# Claude Code Ultrathink 和 Ultracode 区别

用能解决问题的最小模式。更多推理不等于更好。

> 最后核查：2026 年 7 月 7 日。Claude Code 的推理模式、工作流名称和界面标签可能变化。这里是实用判断指南，最终以你当前 CLI 行为为准。

## 快速答案

| 选择         | 适合场景                           | 不适合场景                   |
| ------------ | ---------------------------------- | ---------------------------- |
| `ultrathink` | 单个难推理步骤、架构问题、调试假设 | 简单编辑或明显命令           |
| Ultracode    | 仓库级任务、审计、迁移、多文件实现 | 小改动、目标不清、低价值探索 |
| Plan Mode    | 编辑前先看方案                     | 已经明确知道要改什么         |

## 什么时候用 Ultrathink

问题主要卡在推理时，用 `ultrathink`：

- 两种架构方案取舍
- 隐蔽 bug 的调试
- 风险迁移的方案设计
- 检查某个修复是否有副作用

它不能替代测试、源码检查或官方文档。

## 什么时候用 Ultracode

任务足够大，才考虑 ultracode：

- 仓库级审计
- 多页面 SEO 清理
- 框架迁移
- 多文件重构并验证
- 需要连续排查的构建或部署失败

如果只是改一行文案，通常不值得用。

## Plan Mode 的位置

Plan Mode 解决的是另一个问题：Claude 是否应该先检查和规划，再动手改。认证、支付、重定向、schema、大规模内容迁移这类场景，Plan Mode 更有价值。
