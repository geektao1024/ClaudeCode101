## 快速结论

按你的搜索意图进入：

- **安装 Claude Code：** 先看安装 FAQ；如果失败，再进完整安装教程和故障排查。
- **Windows 配置：** 先判断用原生 Windows 还是 WSL，再处理 PATH、Shell 和包管理器。
- **价格和用量：** 把价格、限制、`/model`、`/usage`、`/clear`、`/compact` 放在一起看。
- **项目记忆：** 写大段项目说明前，先看 CLAUDE.md 应该写什么。
- **Plan Mode：** 多文件改动、迁移、价格页、重定向、登录、支付等高风险任务先规划。
- **MCP servers：** 一次只接一个 server，并收紧工具权限。
- **自动化：** 重复提示词用自定义命令；大规模编排任务看 Dynamic Workflows；脚本化运行看 headless；程序化集成看 Agent SDK/API 文档。

FAQ 页面负责回答第一个问题，链接页负责帮你真正执行。

## 安装和首次运行问题

安装问题通常拆成两个决策：操作系统和认证方式。

| 你在问什么                     | 先看                                                                      | 需要验证                                      |
| ------------------------------ | ------------------------------------------------------------------------- | --------------------------------------------- |
| “现在该怎么安装 Claude Code？” | [如何安装 Claude Code](/zh/faqs/how-to-install-claude-code)               | `claude --version`、Shell PATH、登录方式。    |
| “为什么 Windows 不一样？”      | [Windows 安装指南](/zh/faqs/how-to-install-claude-code-on-windows)        | 原生终端还是 WSL、包管理器和 PATH。           |
| “为什么 `claude` 运行不了？”   | [command not found 修复](/zh/troubleshooting/claude-command-not-found)    | binary 位置和当前 Shell 加载的 PATH。         |
| “为什么登录失败？”             | [登录排障](/zh/troubleshooting/claude-code-login-or-authentication-error) | 浏览器登录、API key、组织席位、本地凭据状态。 |

CLI 没安装好、PATH 不通、或还没完成认证前，不要急着排查价格、MCP 或 Plan Mode。

## 价格、用量和成本问题

Claude Code 价格问题容易混，因为订阅用量和 API 计费是两套不同表面。

| 问题                                    | 实用答案                                                                                       |
| --------------------------------------- | ---------------------------------------------------------------------------------------------- |
| “Claude Code 免费吗？”                  | 计划可用性会变化，先查官方 pricing 和自己的账号。有些工作流可能需要付费套餐或 API 计费。       |
| “该选 Pro、Max、Team 还是 Enterprise？” | 看编码频率、团队协作、管理/安全要求，以及是否需要组织级控制。                                  |
| “为什么触发限制？”                      | 检查登录方式、当前模型、上下文大小、长会话，以及你是订阅用量还是 API metering。                |
| “怎么让额度更耐用？”                    | 无关任务用 `/clear`，长会话用 `/compact`，模型选择看 `/model`，prompt 尽量指向文件和具体问题。 |
| “怎么追踪成本？”                        | 订阅用户看 plan usage；API 用户看 `/usage`、`/cost`、Console 或云厂商账单页。                  |
| “为什么 Opus 4.8 变慢或变贵？”          | 先查 effort、ultracode、dynamic workflows、上下文大小和 Fast mode，不要直接归咎于模型。        |

做预算时，先看 [Claude Code 多少钱？](/zh/faqs/how-much-is-claude-code)，再看 [Claude Code 用量限制](/zh/faqs/claude-code-usage-limits)。

## 如何使用这个 FAQ

FAQ 页面负责第一个答案，不替代完整手册。好的问题页应该做到：

1. 开头几行直接回答问题。
2. 说明哪些信息变化快、必须核查。
3. 给出下一步可执行动作。
4. 链接到更深的教程或机制页。
5. 链接到官方来源和账号内真实信息。

做生产决策前，请核查：

- 当前套餐和账单规则是否变化。
- Claude Code 当前用的是哪个账号、组织或 API key。
- `/model` 当前模型是否正确。
- 当前任务适合交互运行、Plan Mode、headless mode，还是 Agent SDK。
- MCP 和工具权限是否足够收敛。
- 你看的页面是否有近期 “最后核查” 日期。
