## 大多数开发者应该怎么选

| 开发者类型                                | 选择       | 原因                                            |
| ----------------------------------------- | ---------- | ----------------------------------------------- |
| 新 Claude Code 用户                       | Pro        | 包含 Claude Code，是最低成本的正式起点。        |
| 经常写代码的个人开发者                    | Max 5x     | 能减少中断，不用一上来买最高个人档。            |
| 每天在大仓库里重度使用的个人开发者        | Max 20x    | 个人方案里用量最高。                            |
| 多人创业团队                              | Team       | 统一账单、SSO、连接器控制和用量可见性开始重要。 |
| 企业级推广                                | Enterprise | 合规、审计、花费控制和组织策略比单价更重要。    |
| 自动化、CI、agent、产品化 coding workflow | API        | 需要按量计费、日志、限额和基础设施级控制。      |

不要在这些情况下第一时间升级：上下文太大、误走 API key、日志太长、prompt 太泛、或把昂贵模型长期用于普通修改。

## 订阅用量和 API 计费的区别

比较价格前，先确认当前会话到底走哪条计费路径。

| 当前路径                              | 用量含义                                   | 如何核对                             | 成本风险                                        |
| ------------------------------------- | ------------------------------------------ | ------------------------------------ | ----------------------------------------------- |
| Pro 或 Max 通过 `/login` 使用         | 订阅内用量，并和 Claude 其他产品表面共享。 | `claude`、`/status`、账号计划页。    | 长时间编码会话可能遇到 reset 窗口。             |
| Team 或 Enterprise 通过 `/login` 使用 | 组织席位、策略、用量池和管理员控制。       | 管理后台、组织分析、计划设置。       | 用户可能不知道自己处在哪个席位或策略下。        |
| `ANTHROPIC_API_KEY`                   | 按量 API 计费。                            | shell 环境变量检查、Claude Console。 | 脚本循环、大上下文或 MCP 输出可能造成意外费用。 |
| Bedrock、Vertex、Microsoft Foundry    | 云厂商 token 计费、配额和地区规则。        | 云控制台和账单报表。                 | 云厂商价格和一方 Claude API 可能不同。          |

如果你觉得 Claude Code 突然变贵了，先查认证，不要先升级：

```bash
env | grep ANTHROPIC_API_KEY
claude
/status
/usage
```

Anthropic 支持文档明确提醒：如果系统里设置了 `ANTHROPIC_API_KEY`，Claude Code 可能使用 API 计费，而不是你的订阅内用量。

## Usage Limits 不是 Context Limits

| 限制类型              | 含义                                                       | 最佳处理方式                                                |
| --------------------- | ---------------------------------------------------------- | ----------------------------------------------------------- |
| Usage limit           | 当前计划或组织在某个窗口内的可用量用完。                   | 等 reset、升级、启用 usage credits，或降低模型/上下文消耗。 |
| Weekly limit          | Max 计划有分配给账号的周级 reset 时间。                    | 在 Settings > Usage 看 reset 时间，围绕窗口安排长任务。     |
| Rate limit            | 请求或 token 速度过快。                                    | 降低并发、稍后重试，或申请组织限额。                        |
| Context length limit  | 会话携带了太多历史、文件、日志或 MCP 输出。                | 用 `/compact`、`/clear`，缩小文件范围，裁剪日志。           |
| Spend 或 credit limit | API credits、auto-reload、云厂商配额或工作区上限挡住运行。 | 先查账单，再决定是否继续。                                  |

## 成本控制工作流

1. 运行 `/status`，确认当前是订阅还是 API 计费。
2. 运行 `/model`，选择能完成任务的最低成本模型。
3. 涉及 migration、billing、auth、公开 SEO 页面、数据库、MCP 权限、大重构时，先用 Plan Mode。
4. 指定目标文件路径，不要让 Claude 扫整个仓库。
5. 保持 `CLAUDE.md` 简短，只放每次会话都值得加载的规则。
6. 切换任务用 `/clear`，长任务继续用 `/compact`。
7. MCP server 只保留当前任务需要的。
8. API 脚本、CI、headless agents 运行前，先设置 spend limits。

## 价格常见问题

### Claude Code 包含在 Claude Pro 里吗？

包含。Claude 官方公开价格页把 Claude Code 列在 Pro 功能中。但这不等于无限使用。

### Max 对 Claude Code 值得买吗？

如果你已经做好上下文管理，但真实编码工作仍频繁被 Pro 限制打断，Max 值得考虑。它不能解决 API key 误计费、prompt 太泛、日志太大或普通任务一直用 Opus 的问题。

### Max 和 API 计费怎么选？

人坐在终端或 IDE 里交互式使用，优先 Max。CI、headless agents、脚本、产品自动化、需要基础设施级花费控制的场景，用 API。

### 怎么避免意外 API 扣费？

检查 `ANTHROPIC_API_KEY`，用 `/status` 确认路径；如果想留在 Pro/Max 订阅内，拒绝 API credits，并检查 Claude Console auto-reload。

### 为什么大仓库里 Claude Code 更容易贵？

大仓库会增加文件读取、项目上下文、日志、MCP 输出和迭代长度。明确目标文件的 prompt 通常比“检查整个仓库”便宜得多。

### Claude Code 有学生折扣吗？

官方公开价格页列出了面向机构的 Education plan，包含 student and faculty access，但没有列出个人自助购买的 Claude Pro 学生折扣。如果你是学生，现实路径是确认学校是否提供 Claude for Education。不要默认相信优惠券网站里的 “Claude student discount”，也不要默认它包含 Claude Code。

### Claude 有 trial 吗？

Claude 有 Free plan，适合基础体验，可以把它当成类似 trial 的入口。但不要把正式 Claude Code 终端工作流建立在 Free 上。要用 Claude Code，先在账号里确认计划和计费路径，再在长会话前检查 `/status`。
