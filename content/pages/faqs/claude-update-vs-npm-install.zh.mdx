# Claude update 和 npm install 怎么选

短答案：用最初安装 Claude Code 的同一个渠道更新。

> 最后核查：2026 年 7 月 7 日。npm `latest` dist-tag 为 `2.1.202`，`stable` dist-tag 为 `2.1.195`。更新生产环境前，仍然要核对官方 changelog 和本机安装方式。

## 快速答案

| 当前 `claude` 来源     | 推荐更新方式                                      |
| ---------------------- | ------------------------------------------------- |
| Claude Code 原生安装器 | `claude update`                                   |
| npm 全局包             | `npm install -g @anthropic-ai/claude-code@latest` |
| Homebrew               | `brew upgrade claude-code`                        |
| WinGet                 | `winget upgrade Anthropic.ClaudeCode`             |
| apt                    | `sudo apt update && sudo apt install claude-code` |
| dnf                    | `sudo dnf upgrade claude-code`                    |
| apk                    | `sudo apk update && sudo apk add -u claude-code`  |

## 为什么 `npm update -g` 可能不是最新版

`npm update -g @anthropic-ai/claude-code` 会在 npm 已知的版本范围内更新。如果旧安装被版本范围限制，它不一定会跳到最新的 `latest` tag。

如果你明确要 npm 最新版，用：

```bash
npm install -g @anthropic-ai/claude-code@latest
```

然后验证：

```bash
claude --version
npm view @anthropic-ai/claude-code dist-tags --json
```

## 先判断安装方式

更新前先跑：

```bash
which claude
claude --version
npm list -g @anthropic-ai/claude-code --depth=0
```

如果 `which claude` 指向 npm 全局目录，并且 `npm list -g` 能看到这个包，就用 npm 更新。否则优先按原生安装器或系统包管理器更新。

## 安全更新清单

1. 用 `claude --version` 看当前版本。
2. 阅读 [Claude Code changelog](https://docs.anthropic.com/en/release-notes/claude-code)。
3. 使用同一个安装渠道更新。
4. 重启 shell、IDE、终端 agent 或 CI runner。
5. 原生 CLI 或 auto-updater 在更新后异常时，运行 `claude doctor`。
6. 再跑 `claude --version` 和一个小测试提示词。

如果搜索意图是 `Claude Code release notes` 或 `claude doctor command`，不要只回答一个安装命令。用户真正需要知道的是：版本变了什么、当前 CLI 来自 npm、原生安装器、Homebrew、WinGet、apt、dnf 还是 apk，以及这次更新有没有影响 MCP、hooks、permissions 或模型行为。
