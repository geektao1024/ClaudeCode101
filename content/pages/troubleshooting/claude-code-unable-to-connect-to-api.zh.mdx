# Claude Code API 连接失败排查

如果 Claude Code 提示 `unable to connect to Anthropic services`、`can't reach Claude`、请求超时，或出现 SSL certificate 问题，先把它当成连接链路问题排查。原因可能是本地网络、公司代理、VPN 路由、证书信任、服务状态、认证状态，或错误的 base URL。

> 最后核查：2026 年 7 月 8 日。不同 Claude Code 版本和 provider 的错误文案会变化。重装前先看完整错误、当前 provider 和真实网络路径。

## 先把错误归到对应层级

| 错误文案或搜索词                                    | 最可能的层级                              | 先检查                            | 更好的第一步                       |
| --------------------------------------------------- | ----------------------------------------- | --------------------------------- | ---------------------------------- |
| `unable to connect to Anthropic services`           | 网络、代理、VPN、DNS 或服务状态。         | 状态页和同一终端的网络访问。      | 先换网络测试，不要先改凭证。       |
| `can't reach Claude error`                          | 浏览器或终端到 Claude endpoint 不通。     | VPN 分流和代理变量。              | 对比浏览器访问和终端访问。         |
| `Claude Code API error: 500`                        | provider 或 gateway 返回内部错误。        | provider 状态和请求路径。         | 用小 prompt 稍后重试，先不要重装。 |
| `Claude authorization failed internal server error` | Auth 服务、账号会话或 provider 认证链路。 | `/status`、`/doctor`、登录状态。  | 先确认不是服务事故，再重新登录。   |
| `Claude internal server error`                      | 远端服务错误，有时和账号或 gateway 有关。 | 状态页和 provider 控制台。        | 记录准确时间和请求路径。           |
| `Claude Cowork API error: connection error`         | 特定产品 connector、gateway 或网络链路。  | 当前 base URL 和 connector 设置。 | 校验配置的 endpoint 和凭证。       |

## 先检查这些

1. 查看 Claude 服务状态，或你正在使用的 provider 状态页是否有故障。
2. 在同一个终端环境里发起一个简单网络请求，确认终端能访问外部服务。
3. 如果使用代理，检查 `HTTP_PROXY`、`HTTPS_PROXY` 和证书设置。
4. 如果使用自定义网关或 provider，确认 `ANTHROPIC_BASE_URL` 和相关凭证。
5. Claude Code 能打开时运行 `/doctor`。`claude doctor` 只用于安装或更新后的 CLI / auto-updater 健康检查，不是完整 API 状态测试。
6. 重装前，先换一个网络环境测试一次。

有用的 shell 检查：

```bash
claude --version
env | grep -E 'ANTHROPIC|HTTP_PROXY|HTTPS_PROXY|SSL_CERT_FILE|NODE_EXTRA_CA_CERTS'
```

## 常见原因

公司网络可能会做 TLS 检查、拦截下载或 API 域名，或者要求使用自定义 CA 证书。VPN 也可能让浏览器和终端走不同线路。云服务商配置下，错误可能来自 Bedrock、Vertex、Foundry 或自定义 gateway，而不一定是 Anthropic 本身。

如果设置了 `ANTHROPIC_BASE_URL`，你已经不在测试默认 Anthropic endpoint。一个坏掉的 gateway 会让 Claude Code 看起来像服务不可用，即使 Anthropic 本身正常。如果取消自定义 base URL 后同一条 prompt 恢复，就应该修 gateway 链路，而不是重装 CLI。

## 什么时候需要进一步处理

记录完整错误、操作系统、Shell、代理或 VPN 状态、当前凭证路径，以及换网络后是否恢复。如果 Claude Code 能启动，运行 `/doctor`；如果问题刚好发生在安装或更新后，再在终端运行 `claude doctor`。
