Claude Code 登录问题通常来自本地会话过期、账号或组织权限、浏览器 OAuth 回调失败、不支持的地区，或者使用 Bedrock、Vertex、Foundry 时云服务凭证没有正确加载。

## 先检查这些

1. 如果 Claude Code 能启动，先运行 `/login` 重新登录。
2. 如果看到 OAuth 错误或 `403 Forbidden`，确认账号、组织和地区是否可用。
3. 如果在 SSH、WSL 或容器里登录失败，按提示手动复制浏览器回调 URL。
4. 如果使用 API、Bedrock、Vertex 或 Foundry，检查环境变量和云服务凭证。
5. 用 `/doctor` 或 `claude doctor` 区分登录问题和安装问题。

## 什么时候需要进一步处理

如果同一个账号在浏览器可用，但 CLI 里一直失败，记录完整错误、安装方式、Shell、操作系统，以及是否使用代理或 VPN。
