## 核心机制

- [CLAUDE.md](/zh/mechanics/claude-md) - 项目上下文、常用命令、编码规范和交付规则。
- [Plan Mode](/zh/mechanics/plan-mode) - 改代码前先审查实现方案。
- [Headless Mode](/zh/faqs/claude-code-headless-mode) - 用于自动化和 CI 的可重复提示词执行。
- [Slash Commands](/zh/mechanics/slash-commands) - 把重复提示词变成可复用命令。
- [Hooks](/zh/mechanics/hooks) - 添加工作流自动化和护栏。
- [Subagents](/zh/mechanics/subagents) - 为重复的专业角色配置专门代理。
- [Dynamic Workflows](/zh/mechanics/dynamic-workflows) - 用于大规模审查、迁移和交叉验证研究的后台编排。
- [Ultracode](/zh/mechanics/ultracode) - 用 `xhigh` 加 workflow 判断处理有明确范围的困难会话。
- [上下文管理](/zh/mechanics/context-management) - 减少噪声上下文和无效消耗。

## 推荐阅读路径

1. 先安装 Claude Code，并用小项目跑通第一次会话。
2. 补一个聚焦的 `CLAUDE.md`。
3. 配置工具权限和 GitHub CLI。
4. 大改动前学会 Plan Mode。
5. 只有服务真实工作流时再添加 MCP server。
6. 能清楚收窄任务后，再学习 dynamic workflows 和 ultracode。
7. 基础稳定后，再学习上下文管理和 Headless Mode。
