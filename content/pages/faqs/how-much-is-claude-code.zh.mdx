# Claude Code 多少钱？

Claude Code 没有一个单独统一价格。它包含在符合条件的 Claude Pro、Max、Team、Enterprise 等订阅中，也可以通过 Console 或云厂商 API 计费使用。

> 最后核查：2026 年 7 月 7 日。这是快速回答。购买或改计费方式前，请看 [完整价格指南](/zh/pricing)。

## 短答案

| 你的情况                           | 可能的计费路径        | 实际结论                                                             | 付费前检查                                |
| ---------------------------------- | --------------------- | -------------------------------------------------------------------- | ----------------------------------------- |
| 只有免费 Claude 账号               | 不是 Claude Code 方案 | Free 不适合正式 Claude Code 终端工作流。                             | 在账号里确认是否有 Claude Code access。   |
| 个人学习或轻量开发                 | Pro                   | 官方公开价显示 Pro 年付折算 $17/月，或月付 $20，并包含 Claude Code。 | 用量会和 Claude 其他产品表面共享。        |
| 日常写代码，Pro 经常触顶           | Max                   | Max 从 $100/月起，提供 Pro 的 5x 或 20x 用量。                       | 更高用量不等于无限用量。                  |
| 多个开发者需要统一账单             | Team                  | Team 按公开席位价格起步，并包含 Claude Code。                        | 席位类型、年付/月付和管理员策略要看清。   |
| 组织需要 SSO、审计、合规或花费控制 | Enterprise            | Enterprise 结合席位价格和 API rates 计费。                           | 这是 rollout 决策，不是个人升级。         |
| CI、脚本、agent、产品自动化        | API / 云厂商          | 按模型 token 和功能用量计费。                                        | 无人值守任务运行前必须设置 spend limits。 |

## 最大的价格误区

不要混淆订阅用量和 API 计费。通过 Pro、Max、Team 或 Enterprise 登录时，Claude Code 可以使用订阅额度。如果 `ANTHROPIC_API_KEY` 或云厂商凭据生效，终端会话可能走 API 计费。

长任务前先检查：

```text
/status
/model
```

同时检查 shell 里是否有 API 凭据，不要默认认为当前会话一定在订阅内。

## 什么情况不该先升级

- 如果真正问题是粘贴了巨大日志或 prompt 太泛，不要先升级。
- 没用 `/status` 排除 API 误计费前，不要先升级。
- 无人值守脚本不要用 Max 解决，应该用 API 计费加 spend caps。
- 不要把任何计划当成无限使用。
